Handler stack manipulation

A stack of message handlers is provided to make it easy to install a different message handler appropriate to a new application context, and then revert to the previous handler upon exiting that context. At startup a single default handler resides on the stack - it is an error to try to pop this.

Threading:
It is not safe to manipulate the handler stack from concurrent threads.

Detailed Description

The MessageHandler class should be used for all logging within IECore code, and code using IECore. It provides a uniform interface for outputting messages, with the possibility to implement multiple message handlers appropriate to specific application contexts.
